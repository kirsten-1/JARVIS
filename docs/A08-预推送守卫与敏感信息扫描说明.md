# A08 预推送守卫与敏感信息扫描说明

## 目标

在阶段性 `git push` 前，统一执行最低安全与可用性校验，避免：

- `.env` 误入 Git
- 明文密钥样式字符串进入仓库
- 关键链路冒烟未通过就推进版本节点

## 交付内容

- 新增脚本：`scripts/a08_prepush_guard.sh`
- 自动生成报告：`docs/reports/a08_prepush_<timestamp>.md`

## 默认校验项

- env 跟踪检查：禁止 `.env` / `.env.prod` 被 git 跟踪
- 密钥样式扫描：扫描 tracked 文件中的可疑字面量
- `aiservice` 健康预检（当启用 `a04` 或 `m13+PROVIDER=aiservice` 时自动执行）
- `m11_smoke.sh`
- `a04_smoke.sh`

可选：

- `m13_smoke.sh`（`RUN_M13_SMOKE=true`）

## 使用方式

## 1) 默认执行

```bash
./scripts/a08_prepush_guard.sh
```

## 2) 附加 m13 验收

```bash
RUN_M13_SMOKE=true PROVIDER=aiservice MODEL=MiniMax-M2.5 ./scripts/a08_prepush_guard.sh
```

## 3) 临时只做安全检查（跳过 smoke）

```bash
RUN_M11_SMOKE=false RUN_A04_SMOKE=false ./scripts/a08_prepush_guard.sh
```

## 输出与退出码

- 成功：退出码 `0`
- 失败：退出码 `1`（报告中给出 failed 项）

报告包含：

- 校验结果表
- env 检查与密钥扫描原文
- `aiservice` 健康检查原文（若执行）
- Git 工作区状态
- smoke 输出日志（若执行）

说明：

- 若 `aiservice` 健康预检失败，`a04/m13` 会标记为 `blocked`，并在建议区给出可直接执行的排查命令。
- 当配置里是 `host.docker.internal` 时，脚本会自动转换为 `localhost` 做主机侧健康检查。

## 建议流程

1. `./scripts/a08_prepush_guard.sh`
2. 阅读 `docs/reports/a08_prepush_*.md`
3. 若成功，再执行阶段性 `git push origin main`
