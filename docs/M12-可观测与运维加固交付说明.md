# M12 可观测与运维加固交付说明

## 1. 目标

M12.1 聚焦“可观测可运维”：

- 暴露标准 Prometheus 指标端点。
- 一键拉起 Prometheus + Grafana 监控栈。
- 预置基础看板，覆盖网关 QPS、成功率、延迟、缓存与 JVM 指标。
- 预置 Prometheus 告警规则，覆盖可用性、错误率、时延与 JVM 内存风险。

## 2. 交付清单

- 指标暴露：
  - `pom.xml`（新增 `micrometer-registry-prometheus`）
  - `src/main/resources/application.yml`
  - `src/main/resources/application-prod.yml`
  - `src/main/java/com/bones/gateway/security/SecurityConfig.java`
- 监控编排：
  - `docker-compose.observability.yml`
  - `monitoring/prometheus/prometheus.yml`
  - `monitoring/prometheus/alerts.yml`
  - `monitoring/grafana/provisioning/datasources/datasources.yml`
  - `monitoring/grafana/provisioning/dashboards/dashboards.yml`
  - `monitoring/grafana/dashboards/jarvis-overview.json`
- 运维脚本：
  - `scripts/m12_obs_up.sh`

  - `scripts/m12_obs_smoke.sh`
- 环境模板：
  - `.env.prod.example`（新增 Prometheus/Grafana 端口与管理员账号变量）

## 3. 使用方式

前置条件：
- M11 生产编排已可正常运行（至少 gateway 可用）。
- 本地已准备 `.env.prod`。

启动监控栈：

```bash
./scripts/m12_obs_up.sh
```

访问地址：
- Prometheus: `http://localhost:${PROMETHEUS_PORT:-19090}`
- Grafana: `http://localhost:${GRAFANA_PORT:-13000}`

Grafana 默认账号（可在 `.env.prod` 覆盖）：
- 用户名：`${GRAFANA_ADMIN_USER:-admin}`
- 密码：`${GRAFANA_ADMIN_PASSWORD:-admin}`

冒烟（包含 scrape 与规则加载校验）：

```bash
./scripts/m12_obs_smoke.sh
```

停止仅监控组件：

```bash
./scripts/m12_obs_down.sh
```

停止全栈（应用 + 监控）：

```bash
./scripts/m12_obs_down.sh --all
```

停止并清理卷：

```bash
./scripts/m12_obs_down.sh --volumes
```

## 4. 关键说明

- Prometheus 通过 `gateway:8080/actuator/prometheus` 抓取指标。
- Grafana 已预置 `Prometheus` 数据源与 `Jarvis Gateway Overview` 看板。
- Prometheus 已预置告警组 `jarvis-gateway.rules`，包含：
  - `JarvisGatewayDown`
  - `JarvisHigh5xxRate`
  - `JarvisHighP95Latency`
  - `JarvisJvmHighHeapUsage`
- 当前实现将 `/actuator/prometheus` 设为匿名可访问，生产建议通过网段隔离或网关层 ACL 进行限制。

## 5. 验收标准

- `m12_obs_up` 后，Prometheus 与 Grafana 健康检查通过。
- `m12_obs_smoke` 返回 `SUCCESS`。
- Prometheus `/api/v1/rules` 可看到 `jarvis-gateway.rules` 告警组。
- Grafana 中能看到 `Jarvis Gateway Overview` 默认看板并出现数据。
