# A20 收官决策与交付快照说明

## 目标

在 A19（证据包审计）之后，补齐最终收官层：

- 基于 A17/A18/A19 给出 `go/hold` 决策
- 生成收官快照与收官包
- 默认保持“仅归档，不自动 release”的策略

## 交付内容

文件：

- `scripts/a20_final_closeout.sh`
- `.github/workflows/a20-final-closeout.yml`

输出：

- 收官报告：`docs/reports/a20_final_closeout_<timestamp>.md`
- 收官快照：`docs/reports/bundles/a20_closeout_snapshot_<timestamp>.json`
- 收官归档包：`docs/reports/bundles/a20_closeout_package_<timestamp>.tar.gz`

## 1) A20 收官脚本

脚本：

- `scripts/a20_final_closeout.sh`

能力：

1. 自动读取最新：
   - `a17_readiness_bundle_*.md`
   - `a18_release_candidate_bundle_*.md`
   - `a19_bundle_audit_*.md`
2. 汇总质量状态并输出最终决策：
   - `decision=go`：仅当质量通过且 `ALLOW_RELEASE=true`
   - `decision=hold`：其余情况
3. 生成 JSON 快照（commit/branch/gates/decision）。
4. 生成收官包（含关键报告、快照、A18 bundle 引用）。

关键变量：

- `ALLOW_RELEASE`：默认 `false`（默认不自动 release）
- `STRICT`：默认 `true`
- `RUN_A19_FIRST`：默认 `false`（可选先跑一次 A19）

## 2) A20 CI workflow

文件：

- `.github/workflows/a20-final-closeout.yml`

流程：

1. 启动 `mysql + redis + gateway`
2. 执行 A08 ~ A19 全链路
3. 执行 A20 收官脚本
4. 上传 A08~A20 报告及收官快照/收官包 artifact

触发：

- 手动 `workflow_dispatch`
- 每周定时巡检（`cron: 30 4 * * 1`，UTC）

## 使用方式

## 1) 默认执行（保持 hold）

```bash
./scripts/a20_final_closeout.sh
```

## 2) 允许给出 go 决策（仅当质量全 pass）

```bash
ALLOW_RELEASE=true ./scripts/a20_final_closeout.sh
```

## 建议

1. 当前阶段推荐继续使用默认 `ALLOW_RELEASE=false`，保持“全项目收官后再 release”策略。
2. 当 JARVIS 全局准备完毕时，再由收官窗口显式开启 `ALLOW_RELEASE=true`。
