# M4.6 多厂商路由交付说明

## 1. 目标

- 在不改网关对外接口路径的前提下，支持多厂商 AI 动态切换。
- 支持请求级别的 `provider/model` 指定。
- 支持主 provider 故障时自动 fallback 到备 provider。

## 2. 本次交付

## 2.1 请求模型扩展

- `AiChatRequestDto` 新增字段：
  - `provider`：目标厂商（可选）
  - `model`：目标模型（可选）
- `AiChatRequest` 同步新增上述字段。

文件：
- `src/main/java/com/bones/gateway/dto/AiChatRequestDto.java`
- `src/main/java/com/bones/gateway/integration/ai/model/AiChatRequest.java`

## 2.2 多 provider 配置模型

- `AiServiceProperties` 新增：
  - `default-provider`
  - `fallback-enabled`
  - `fallback-provider`
  - `providers.*`（每个 provider 的协议、URL、路径、Key 等）
- 保留 legacy 单 provider 字段兼容旧配置。

文件：
- `src/main/java/com/bones/gateway/config/AiServiceProperties.java`
- `src/main/resources/application-dev.yml`
- `.env.example`

## 2.3 路由与协议适配

- `WebClientAiServiceClient` 增加：
  - provider 解析（请求字段优先，metadata 次之，最后 default）
  - provider 校验（未配置/禁用/配置非法）
  - OpenAI-compatible 协议适配
  - Gemini 协议适配（含 `{model}` 路径占位符、query key）
  - 主 provider 失败后的 fallback 调用

文件：
- `src/main/java/com/bones/gateway/integration/ai/WebClientAiServiceClient.java`

## 2.4 错误码扩展

- 新增：
  - `AI_PROVIDER_NOT_FOUND`
  - `AI_PROVIDER_DISABLED`
  - `AI_PROVIDER_CONFIG_INVALID`

文件：
- `src/main/java/com/bones/gateway/common/ErrorCode.java`

## 2.5 测试扩展

- `WebClientAiServiceClientTest` 新增/覆盖：
  - 指定 provider 路由
  - fallback 生效
  - Gemini 响应解析

文件：
- `src/test/java/com/bones/gateway/integration/ai/WebClientAiServiceClientTest.java`

## 3. 支持的厂商配置模板

- `deepseek`（OpenAI-compatible）
- `gemini`（Gemini 协议）
- `aliyun`（DashScope compatible mode）
- `tencent`（按兼容网关地址配置）
- `glm`（OpenAI-compatible）
- `minimax`（按账户/区域网关配置）
- `local`（默认本地调试）

## 4. 使用方式

请求示例（同步）：

```json
{
  "message": "你好",
  "conversationId": 1,
  "userId": 1001,
  "provider": "deepseek",
  "model": "deepseek-chat",
  "metadata": {
    "requestId": "req-20260228-001"
  }
}
```

如果不传 `provider/model`：
- `provider` 使用 `jarvis.ai.default-provider`
- `model` 使用对应 provider 的默认 `model`
