# A18 候选发布证据包归档说明

## 目标

在 A17（发布就绪总览）基础上，新增“候选发布证据包”能力：

- 自动收集关键质量报告
- 生成可追溯证据包（tar.gz + manifest + checksums）
- 仅做归档与评审输入，不执行 release

## 交付内容

文件：

- `scripts/a18_release_candidate_bundle.sh`
- `.github/workflows/a18-release-candidate-bundle.yml`

输出：

- 汇总报告：`docs/reports/a18_release_candidate_bundle_<timestamp>.md`
- 证据包：`docs/reports/bundles/a18_rc_bundle_<timestamp>.tar.gz`

## 1) A18 证据包脚本

脚本：

- `scripts/a18_release_candidate_bundle.sh`

能力：

1. 采集最新报告（默认 required）：
   - `a08_prepush_*.md`
   - `a11_baseline_*.md`
   - `a13_baseline_diff_*.md`
   - `a15_baseline_trend_*.md`
   - `a16_trend_gate_*.md`
   - `a17_readiness_bundle_*.md`
2. 可选采集 `a14_gate_*.md`。
3. 生成证据包内容：
   - `manifest.txt`
   - `checksums.sha256`
   - `reports/*.md`
4. 支持严格策略：
   - `REQUIRE_A17_PASS=true` 时，A17 非 `pass` 则阻断
   - `STRICT=true` 时，失败返回非 0

关键变量：

- `REPORT_DIR`：默认 `docs/reports`
- `BUNDLE_DIR`：默认 `docs/reports/bundles`
- `REQUIRED_CHECKS`：默认 `a08,a11,a13,a15,a16,a17`
- `OPTIONAL_CHECKS`：默认 `a14`
- `REQUIRE_A17_PASS`：默认 `true`
- `STRICT`：默认 `true`

## 2) A18 CI workflow

文件：

- `.github/workflows/a18-release-candidate-bundle.yml`

流程：

1. 启动 `mysql + redis + gateway`
2. 执行 A08 轻量守卫
3. 多轮 A11 样本
4. 执行 A13 + A15 + A16 + A17
5. 执行 A18 证据包归档
6. 上传 A08~A18 报告与 tar.gz artifact

触发：

- 手动 `workflow_dispatch`
- 每周定时巡检（`cron: 50 3 * * 6`，UTC）

## 使用方式

## 1) 本地归档（默认）

```bash
./scripts/a18_release_candidate_bundle.sh
```

## 2) 允许 A17 非 pass 也继续打包（不推荐）

```bash
REQUIRE_A17_PASS=false STRICT=false ./scripts/a18_release_candidate_bundle.sh
```

## 建议

1. 在“暂不 release”的阶段，持续用 A18 产出可审计证据包。
2. 到全项目收官时，再基于最近 A18 包进行最终 release 决策。
