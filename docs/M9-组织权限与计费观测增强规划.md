# M9 组织权限与计费观测增强规划

## 1. 目标

在 M1-M8 已完成基础上，M9 聚焦“可商用治理能力”三件事：

- 组织级权限：从用户级升级为 workspace/project/member 模型。
- 真实计费口径：对齐各厂商 usage 字段，提升成本准确性。
- 运营可观测：补齐业务指标与看板接口，支持运营决策。

## 2. 范围

## 2.1 组织与权限模型

- 新增实体与关系：
  - `workspace`
  - `workspace_member`
  - `project`（可选，若首版范围紧张可延后）
- 权限角色：
  - `OWNER`
  - `ADMIN`
  - `MEMBER`
  - `VIEWER`
- 核心规则：
  - 会话与消息默认归属 workspace。
  - 非成员不可访问 workspace 下资源。
  - 管理角色可管理成员与配额策略。

## 2.2 真实账单口径

- 对接 provider 原始 usage 字段（prompt/completion/total）。
- 建立统一 usage 映射层，替代纯估算口径。
- 对每次调用落账本明细，支持按 `workspace/provider/model/day` 聚合。
- 保留估算口径作为 fallback，并记录来源标识（`actual/estimated`）。

## 2.3 运营指标与看板

- 指标补齐：
  - 对话成功率
  - provider 故障率
  - fallback 命中率
  - TTFT 分布（P50/P90/P99）
  - workspace 日成本与配额消耗率
- 增加聚合查询 API：
  - `GET /api/v1/metrics/overview`
  - `GET /api/v1/metrics/providers`
  - `GET /api/v1/billing/workspaces/daily`

## 3. 交付拆分（建议）

## 3.1 M9.1 组织模型与权限收敛

- 数据库迁移：workspace/member 基础表。
- 鉴权上下文扩展：支持 workspace 作用域。
- 会话/消息接口改造：必须带 workspace 或从 token 上下文解析。
- 单测：权限越权、角色边界、跨 workspace 隔离。

## 3.2 M9.2 计费精度升级

- provider usage 解析器（多厂商适配）。
- 账本明细存储与聚合查询。
- fallback 到估算口径的可观测日志与指标。
- 单测：各 provider usage 映射、聚合口径一致性。

## 3.3 M9.3 运营看板接口

- 指标埋点补齐与 Prometheus 暴露。
- 看板查询 API 与分页/时间窗口过滤。
- 前端 M8 增加“运营面板”页（可先做只读展示）。
- 集成测试：接口可用性与关键字段校验。

## 4. 验收标准

- 新建 workspace 后，成员权限边界可被强校验，越权请求返回 403。
- 成本统计可区分 `actual` 与 `estimated`，并按 provider/model 聚合。
- 可通过 API 查询最近 7 天的成功率、TTFT、fallback 命中率与成本曲线。
- 核心用例具备自动化测试，且新增文档可支持新同学独立验收。

## 5. 风险与取舍

- 风险：一次性改造现有 userId 语义，回归面较大。
- 建议：先做“兼容模式”（保留 userId，新增 workspace 透传），再逐步收敛。
- 风险：不同厂商 usage 字段差异大。
- 建议：以统一映射层隔离差异，未覆盖字段先标记 `estimated`。

## 6. 参考顺序

1. 先落库与权限模型（M9.1）
2. 再升级计费口径（M9.2）
3. 最后补运营看板（M9.3）
