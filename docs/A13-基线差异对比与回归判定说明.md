# A13 基线差异对比与回归判定说明

## 目标

把 A11 的单次基线报告升级为“可比较、可判定”的回归能力，用统一规则回答：

- 新版本相对基线是变快还是变慢
- 回归是否达到阻断级别

## 交付内容

文件：

- `scripts/a13_baseline_diff.sh`

能力：

1. 对比两份 A11 报告（默认自动选 `docs/reports` 最新两份）。
2. 逐项比较 P90 指标（health / conversation / message / agent）。
3. 输出差异报告：`docs/reports/a13_baseline_diff_<timestamp>.md`。
4. 回归判定等级：
   - `pass`：整体稳定或提升
   - `warn`：存在回归，但未触发核心阻断条件
   - `fail`：核心指标回归超过阈值
5. 支持 `STRICT=true` 门禁（`fail` 时直接退出非 0）。

## 默认规则

- 稳定区间：`±5%`（`STABLE_PCT_THRESHOLD`）
- 核心回归阈值：`20%`（`MAX_REGRESSION_PCT`）
- 核心指标：`health`、`conversation_list_hot`、`message_list_hot`、`agent_loop`

## 使用方式

## 1) 自动比较最新两份 A11 报告

```bash
./scripts/a13_baseline_diff.sh
```

## 2) 指定 base/candidate 文件

```bash
BASE_REPORT=docs/reports/a11_baseline_20260304_090000.md \
CANDIDATE_REPORT=docs/reports/a11_baseline_20260305_090000.md \
./scripts/a13_baseline_diff.sh
```

## 3) 严格门禁模式

```bash
STRICT=true ./scripts/a13_baseline_diff.sh
```

## 可调参数

- `STABLE_PCT_THRESHOLD`：默认 `5`
- `MAX_REGRESSION_PCT`：默认 `20`
- `EPSILON_MS`：默认 `0.05`（避免极小抖动误判）
- `OUTPUT_REPORT`：自定义输出路径

## 建议流程

1. 跑 A11：`./scripts/a11_baseline.sh`
2. 跑 A13：`./scripts/a13_baseline_diff.sh`
3. 结合 A13 结果决定是否进入发布/下一阶段
