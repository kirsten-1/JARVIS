# A19 证据包审计与索引说明

## 目标

在 A18（证据包归档）之后，增加审计层，回答两个问题：

- 证据包是否完整且未损坏（manifest/checksum/内容一致）
- 最新证据包状态是否可直接进入收官评审

## 交付内容

文件：

- `scripts/a19_bundle_audit.sh`
- `.github/workflows/a19-bundle-audit.yml`

输出：

- 审计报告：`docs/reports/a19_bundle_audit_<timestamp>.md`
- 索引台账：`docs/reports/bundles/a19_bundle_index.md`

## 1) A19 审计脚本

脚本：

- `scripts/a19_bundle_audit.sh`

能力：

1. 扫描 `docs/reports/bundles/a18_rc_bundle_*.tar.gz`（可配置目录）。
2. 解包后校验：
   - `manifest.txt` 存在
   - `checksums.sha256` 存在
   - `reports/*.md` 与校验值一致
3. 读取 bundle 内 `a17_conclusion` 进行审计标记。
4. 输出总览结论 `pass/warn/fail`，并生成可读索引台账。

关键变量：

- `REPORT_DIR`：默认 `docs/reports`
- `BUNDLE_DIR`：默认 `docs/reports/bundles`
- `LIMIT`：默认 `10`（最多审计最近 N 个包）
- `STRICT`：默认 `true`
- `FAIL_ON_WARN`：默认 `false`

## 2) A19 CI workflow

文件：

- `.github/workflows/a19-bundle-audit.yml`

流程：

1. 启动 `mysql + redis + gateway`
2. 执行 A08 + A11 + A13 + A15 + A16 + A17 + A18
3. 执行 A19 审计并输出 index
4. 上传 A08~A19 报告及 bundle artifact

触发：

- 手动 `workflow_dispatch`
- 每周定时巡检（`cron: 15 4 * * 0`，UTC）

## 使用方式

## 1) 默认执行

```bash
./scripts/a19_bundle_audit.sh
```

## 2) 严格审计（warn 也阻断）

```bash
STRICT=true FAIL_ON_WARN=true ./scripts/a19_bundle_audit.sh
```

## 3) 审计临时目录（测试用）

```bash
REPORT_DIR=/tmp/reports BUNDLE_DIR=/tmp/reports/bundles ./scripts/a19_bundle_audit.sh
```

## 建议

1. A19 适合作为“收官前最后一层”证据完整性检查。
2. 只有 A18 和 A19 都稳定通过时，再进入全项目最终 release 决策。
