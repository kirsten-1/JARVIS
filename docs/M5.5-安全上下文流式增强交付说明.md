# M5.5 安全上下文流式增强交付说明

## 1. 目标

本次增强聚焦三类关键能力：

1. JWT + RBAC + 配额/限流（商业可用门槛）
2. 上下文拼装 + Redis 会话缓存（对话质量与成本）
3. 流式聊天落库 + 首 token 时延指标（体验与运营）

## 2. 交付内容

## 2.1 安全能力（JWT + RBAC）

新增组件：
- `security/SecurityConfig`
- `security/JwtAuthenticationFilter`
- `security/JwtTokenService`
- `security/AccessControlService`
- `security/RestAuthenticationEntryPoint`
- `security/RestAccessDeniedHandler`
- `controller/AuthController`（dev/test 发 token）

关键点：
- 所有业务接口默认需要 Bearer Token。
- 支持 `USER/ADMIN` 两种角色。
- 非管理员只能操作自己的会话；管理员可代操作 `userId`。

## 2.2 配额与限流

新增：
- `config/GuardProperties`
- `service/RequestGuardService`

策略：
- 分钟级限流：`per-minute-limit`
- 日配额：`per-day-quota`
- provider/model 日配额：`per-day-provider-model-quota`
- 存储介质：Redis 计数器 + TTL

异常语义：
- `RATE_LIMITED`（429）
- `QUOTA_EXCEEDED`（429）

## 2.3 上下文拼装与 Redis 缓存

新增：
- `config/ConversationContextProperties`
- `service/ConversationContextService`
- `integration/ai/model/AiChatMessage`

能力：
- 从历史消息构建多轮 prompt。
- 支持窗口裁剪（`window-size`）。
- 支持 Redis 缓存（`cache-ttl-seconds`）。
- AI 请求支持 `messages`，不再只发单轮 message。
- 默认标题会话自动命名：首轮完成后将“新会话”替换为输入摘要标题。

## 2.4 流式聊天落库 + 首 token 时延

增强：
- `ConversationController` 新增：
  - `POST /api/v1/conversations/{conversationId}/chat/stream`
  - `GET /api/v1/conversations/streams/{streamId}`（恢复查询）
- `ChatService` 新增流式主链路：
  - 用户消息先落库
  - 调用 AI stream
  - stream 完成后聚合并落库 assistant 消息
- `StreamSessionService` 新增断线恢复快照（status/content/assistantMessageId）
- 指标：
  - `jarvis.chat.stream.first_token.latency`

## 2.5 成本核算（provider/model 维度）

新增：
- `config/BillingProperties`
- `service/TokenEstimator`
- `service/BillingService`
- `controller/BillingController`（`GET /api/v1/billing/daily`）

能力：
- 估算 prompt/completion tokens。
- 按 `userId + date + provider + model` 记录请求数、token 数、估算成本。
- 支持按日查询账本快照（用于成本看板/结算基础）。

## 2.6 兼容与错误码

新增错误码：
- `UNAUTHORIZED`
- `FORBIDDEN`
- `RATE_LIMITED`
- `QUOTA_EXCEEDED`

会话业务错误码保留并继续生效：
- `CONVERSATION_NOT_FOUND`
- `CONVERSATION_ARCHIVED`

## 3. 配置说明

新增环境变量（见 `.env.example`）：
- `JARVIS_SECURITY_*`
- `JARVIS_GUARD_*`
- `JARVIS_CONTEXT_*`
- `JARVIS_BILLING_*`
- `JARVIS_STREAM_*`

默认建议：
- 开发环境先用 `/api/v1/auth/dev-token` 申请测试 token。
- 生产环境禁用 dev-token 入口或切 profile。

## 4. 测试覆盖

新增测试：
- `AccessControlServiceTest`
- `RequestGuardServiceTest`
- `ConversationContextServiceTest`

增强测试：
- `ChatServiceTest`（新增流式落库路径）
- `WebClientAiServiceClientTest`（保持多 provider 路由回归）

## 5. 验收结果

- `mvn -DskipTests compile` 通过
- `mvn test` 通过

## 6. 后续建议

- 将 dev-token 接口下沉到独立 admin 工具服务，避免生产误开。
- 在配额维度增加 `provider/model` 粒度与成本核算。
- 将流式落库扩展为“增量落库 + 客户端断线恢复”。
