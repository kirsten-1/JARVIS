# gateway_pro 项目完整规划与里程碑

## 1. 项目目标

- 构建 Jarvis 第一项目中的 Spring Boot 网关服务。
- 采用分阶段交付：先打通工程与数据层，再实现网关能力、AI 调用、业务 API、缓存与测试交付。

## 1.1 在 Jarvis 全局中的作用

`gateway_pro` 是 Jarvis 的统一 AI 网关与会话编排层，当前承担以下职责：

- 统一入口：对前端/调用方提供一致的 REST + SSE 接口，屏蔽下游厂商差异。
- 安全与治理：提供 JWT 鉴权、RBAC、限流与配额控制，作为业务访问闸门。
- 会话与消息中台：承载会话创建、消息存储、上下文拼装、流式会话恢复。
- 多模型路由层：按 `provider/model` 做动态路由与 fallback，支持多厂商切换。
- 性能与可观测层：提供 Redis 读缓存/热点缓存、TTFT 与调用指标、基础计费统计。

对 Jarvis 后续模块（Agent / Workflow / RAG / Memory）的价值：

- 作为统一接入层复用鉴权、配额、路由、观测能力，避免各模块重复建设。
- 作为“模型调用基础设施”承接后续工具调用、知识检索和编排执行流量。
