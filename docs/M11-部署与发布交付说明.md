# M11 部署与发布交付说明

## 1. 目标

M11 聚焦“可发布、可部署、可回滚”：

- 增加生产部署编排（基于镜像，而非本地 build）。
- 增加镜像发布流程（本地脚本 + GitHub Actions tag 自动发布）。
- 增加生产 profile 配置与最小化冒烟验证脚本。

## 2. 交付清单

- 生产配置：
  - `src/main/resources/application-prod.yml`
  - `.env.prod.example`
- 部署编排：
  - `docker-compose.prod.yml`
  - `scripts/m11_prod_up.sh`
  - `scripts/m11_prod_down.sh`
  - `scripts/m11_smoke.sh`
- 发布能力：
  - `scripts/m11_release.sh`
  - `.github/workflows/release-images.yml`

## 3. 发布流程（镜像）

### 3.1 本地发布

```bash
# 只构建，不推送
./scripts/m11_release.sh v1.0.0

# 构建并推送到 ghcr.io
IMAGE_NAMESPACE=<your_github_org_or_user> PUSH_IMAGES=true ./scripts/m11_release.sh v1.0.0
```

默认镜像名：
- `ghcr.io/<namespace>/jarvis-gateway-pro:<tag>`
- `ghcr.io/<namespace>/jarvis-gateway-console:<tag>`

### 3.2 CI 自动发布

工作流：`.github/workflows/release-images.yml`

触发方式：
- 推送 tag：`v*`（例如 `v1.0.0`）
- 手工触发 `workflow_dispatch`

发布产物：
- `ghcr.io/<owner>/jarvis-gateway-pro:<tag|latest>`
- `ghcr.io/<owner>/jarvis-gateway-console:<tag|latest>`

## 4. 部署流程（生产 compose）

1. 复制生产环境模板：

```bash
cp .env.prod.example .env.prod
```

2. 修改 `.env.prod` 中关键配置（必须项）：
- `GATEWAY_IMAGE`
- `FRONTEND_IMAGE`
- `MYSQL_ROOT_PASSWORD`
- `REDIS_PASSWORD`
- `JARVIS_JWT_SECRET`
- 至少一个可用 AI provider 的启用与 API Key（如 `AI_GLM_ENABLED=true` + `AI_GLM_API_KEY`）

3. 启动：

```bash
./scripts/m11_prod_up.sh
```

4. 冒烟：

```bash
FRONTEND_URL=http://localhost:${FRONTEND_PORT:-80} \
GATEWAY_URL=http://localhost:${GATEWAY_PORT:-8080} \
./scripts/m11_smoke.sh
```

5. 下线：

```bash
./scripts/m11_prod_down.sh
```

清理数据卷：

```bash
./scripts/m11_prod_down.sh --volumes
```

## 5. 关键说明

- `docker-compose.prod.yml` 为独立生产编排文件，直接使用镜像部署（不依赖 dev compose 叠加）。
- 网关运行 `prod` profile（`SPRING_PROFILES_ACTIVE=prod`）：
  - `swagger-ui` 与 `v3/api-docs` 默认关闭。
  - 健康检查详情默认不对外暴露。
- 前端 Nginx 已关闭 `/api` 代理缓冲，流式输出更稳定。

## 6. 回滚建议

- 镜像回滚：将 `.env.prod` 中 `GATEWAY_IMAGE`、`FRONTEND_IMAGE` 回退到旧 tag。
- 配置回滚：保留上一版 `.env.prod.bak`，回滚后执行 `./scripts/m11_prod_up.sh` 重建容器。
