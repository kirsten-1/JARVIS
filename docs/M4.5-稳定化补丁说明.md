# M4.5 稳定化补丁说明

## 1. 背景

M4 已完成 AI 调用链路（同步/流式），但在上线前仍存在工程稳定性风险：
- 错误 HTTP 语义不够准确（业务码有了，HTTP 状态不够区分）。
- POST 请求重试可能导致重复请求副作用。
- 流式数据格式依赖上游细节，兼容性不足。
- AI 调用可观测性不足（缺乏基础指标）。

M4.5 目标是“最小可挽回补丁”，在不改变主业务流程的前提下提升稳定性。

## 2. 补丁内容

## 2.1 错误语义修正

- `BusinessException` 新增 `HttpStatus` 字段：
  - `src/main/java/com/bones/gateway/common/BusinessException.java`
- `GlobalExceptionHandler` 改为按异常携带状态返回：
  - `src/main/java/com/bones/gateway/config/GlobalExceptionHandler.java`

结果：
- AI 超时 -> 504
- AI 不可达 -> 503
- AI 错误响应 -> 502
- 异常路径响应体保留 `traceId`（便于排障）

## 2.2 重试策略收敛

- AI 配置新增 `retry-enabled`，默认 `false`
- `max-retries` 默认改为 `0`
- 文件：
  - `src/main/java/com/bones/gateway/config/AiServiceProperties.java`
  - `src/main/resources/application-dev.yml`
  - `.env.example`

结果：
- 默认不对 POST 做重试，规避重复调用副作用。

## 2.3 请求防重基础字段

- AI 下游调用时新增 `X-Request-Id` 请求头（优先 metadata.requestId，其次 traceId）。
- 文件：
  - `src/main/java/com/bones/gateway/integration/ai/WebClientAiServiceClient.java`

## 2.4 流式输出规范化

- 在 AI client 中统一解析流式 `data:` 行，过滤空行和 `[DONE]`。
- 文件：
  - `src/main/java/com/bones/gateway/integration/ai/WebClientAiServiceClient.java`

## 2.5 观测指标补齐

- 新增基础指标：
  - `jarvis.ai.chat.latency`
  - `jarvis.ai.chat.errors`
  - `jarvis.ai.stream.latency`
  - `jarvis.ai.stream.sessions`
  - `jarvis.ai.stream.errors`
- 文件：
  - `src/main/java/com/bones/gateway/integration/ai/WebClientAiServiceClient.java`

## 2.6 回归测试补齐

- 新增异常状态映射测试：
  - `src/test/java/com/bones/gateway/config/GlobalExceptionHandlerTest.java`
- AI client 新增 stream 规范化测试：
  - `src/test/java/com/bones/gateway/integration/ai/WebClientAiServiceClientTest.java`
- 增加 Mockito 运行稳定化配置（避免 JDK 21 下 inline agent 附加失败）：
  - `src/test/resources/mockito-extensions/org.mockito.plugins.MockMaker`

## 3. 验收结果

- `mvn -DskipTests compile` 通过
- `mvn test` 通过
  - `GatewayApplicationTests` 通过
  - `GlobalExceptionHandlerTest` 通过
  - `WebClientAiServiceClientTest` 通过（含新增 stream 用例）

## 4. 对后续 M5 的价值

- 错误码与 HTTP 状态语义清晰，前端可按状态做兜底策略。
- 避免 AI 重试导致重复生成/重复计费风险。
- 流式解析规则统一，便于后续接入不同 LLM 服务。
- 关键指标已具备，M5 开始即可观察 AI 调用质量。
