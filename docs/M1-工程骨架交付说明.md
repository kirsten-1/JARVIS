# M1 工程骨架交付说明

## 1. M1 目标

M1 的目标是把 `gateway_pro` 从 Maven 模板工程升级为可持续开发的 Spring Boot 网关骨架，包含：

1. 启动入口与包结构。
2. 基础配置与环境变量模板。
3. 健康检查与 Swagger 可访问能力。
4. 可稳定执行的基础测试。

## 2. M1 实际完成内容

### 2.1 启动入口替换

- 删除默认模板类：`src/main/java/com/bones/App.java`
- 新增 Spring Boot 启动类：`src/main/java/com/bones/gateway/GatewayApplication.java`

关键代码：

```java
@SpringBootApplication
public class GatewayApplication {
    public static void main(String[] args) {
        SpringApplication.run(GatewayApplication.class, args);
    }
}
```

### 2.2 包结构骨架预置

在 `src/main/java/com/bones/gateway/` 下预置了以下包（通过 `package-info.java`）：

- `common`
- `config`
- `controller`
- `dto`
- `entity`
- `integration`
- `repository`
- `service`

目的：后续 M2-M7 可以按层直接落代码，不再反复调整目录结构。

### 2.3 主配置文件

新增：

- `src/main/resources/application.yml`
- `src/main/resources/application-dev.yml`
- `.env.example`

关键点：

1. 默认 profile 为 `dev`。
2. 开启了 Actuator（`health`、`info`）。
3. 开启了 springdoc Swagger UI。
4. 预置了 MySQL、Redis、AI Service 地址配置项。

### 2.4 依赖补齐

`pom.xml` 新增：

1. `spring-boot-starter-actuator`（健康检查）。
2. `com.h2database:h2`（测试环境内存数据库，仅 test scope）。

### 2.5 测试可运行能力

新增：

- `src/test/java/com/bones/gateway/GatewayApplicationTests.java`
- `src/test/resources/application-test.yml`

关键点：

1. 测试类使用 `@ActiveProfiles("test")`。
2. 测试 profile 使用 H2，避免本地没启动 MySQL 时 `contextLoads` 失败。

关键代码：

```java
@SpringBootTest
@ActiveProfiles("test")
class GatewayApplicationTests {
    @Test
    void contextLoads() {}
}
```

## 3. 在 IDEA 中如何验证 M1

### 3.1 快速验证（不依赖 MySQL，推荐）

1. 用 IDEA 打开目录：`jarvis-platform/gateway/gateway_pro`。
2. 在右下角确认 Project SDK 是 JDK 21。
3. 在 Maven 工具窗口执行 `Reload All Maven Projects`。
4. 打开 `GatewayApplicationTests`，点击运行测试。
5. 预期结果：
- 测试通过：`Tests run: 1, Failures: 0, Errors: 0`
- 看到 Spring Boot 启动日志，active profile 为 `test`。

### 3.2 启动应用验证（依赖 MySQL）

1. 先启动 MySQL（数据库名 `jarvis`）和 Redis（6379）。
2. 在 IDEA 新建/编辑 Spring Boot Run Configuration：
- Main class: `com.bones.gateway.GatewayApplication`
- Active profiles: `dev`
- Environment variables: 按 `.env.example` 配置
3. 点击运行应用。
4. 启动后验证：
- `http://localhost:8080/actuator/health` 返回 `status=UP`
- `http://localhost:8080/swagger-ui.html` 可打开（当前阶段接口为空属于正常）

## 5. M1 到 M2 的衔接说明

M1 已完成并可验证。进入 M2 前建议准备：

1. 保持 MySQL/Redis 常驻（后续实体、仓储、联调都会依赖）。
2. 确认 `application-dev.yml` 中数据库连接参数与你本机一致。
