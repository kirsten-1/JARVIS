# M7 测试与交付说明

## 1. 目标

M7 聚焦“可验证、可复现、可交付”三件事：

- 测试补强：覆盖 M6 缓存新增能力
- 冒烟验证：提供一条命令可跑的核心链路脚本
- 交付沉淀：补充运行/验收文档，支持新同学独立拉起

## 3. 执行命令

## 3.1 编译检查

```bash
mvn -DskipTests compile
```

## 3.2 M7 关键测试

```bash
mvn -Dtest=ReadCacheServiceTest,HotResponseCacheServiceTest,ChatServiceTest,ConversationServiceTest test
```

说明：全量 `mvn test` 在受限环境可能因 MockWebServer 端口绑定失败（`Operation not permitted`），不属于业务代码回归。

## 4. 冒烟脚本

脚本路径：`scripts/m7_smoke.sh`

默认验证：

- `/actuator/health`
- `/api/v1/auth/dev-token`
- `/api/v1/conversations`（创建）
- `/api/v1/conversations/{id}/messages`（追加+查询）
- `/api/v1/conversations`（列表）

执行：

```bash
./scripts/m7_smoke.sh
```

可选 AI 验证：

```bash
RUN_AI_SMOKE=true PROVIDER=glm MODEL=glm-4.6v-flashx ./scripts/m7_smoke.sh
```

## 5. IDEA 验收步骤

1. 启动 MySQL、Redis。
2. IDEA 启动 `GatewayApplication`（`dev` profile）。
3. 终端执行 `./scripts/m7_smoke.sh`，观察 `SUCCESS`。
4. 打开前端控制台（M8）验证同步/流式。
5. 若验证 M6 缓存，可重复调用会话列表与消息列表，观察响应时间与 Redis key 变化。

## 6. 风险与说明

- 冒烟脚本依赖 `jq`（需本机安装）。
- AI 冒烟依赖有效 provider key 与网络可达。
- 受限沙箱环境可能无法执行需要本地端口监听的测试。
