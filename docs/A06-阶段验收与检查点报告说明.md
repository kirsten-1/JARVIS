# A06 阶段验收与检查点报告说明

## 目标

在关键开发节点（例如 A04/A05 完成后）自动生成一份可追溯报告，便于：

- 自检当前链路是否可用
- 留存阶段证据（验收输出 + compose 状态）
- 决定是否进行阶段性 `git commit/push`

## 交付内容

- 新增脚本：`scripts/a06_checkpoint.sh`
- 报告输出目录：`docs/reports/`

## 使用方式

## 1) 默认执行（跑 m11_smoke + a04_smoke）

```bash
./scripts/a06_checkpoint.sh
```

## 2) 先拉起再验收

```bash
RUN_UP_FIRST=true ./scripts/a06_checkpoint.sh
```

## 3) 仅做基础 smoke

```bash
RUN_A04_SMOKE=false ./scripts/a06_checkpoint.sh
```

## 报告内容

报告文件示例：

- `docs/reports/a06_checkpoint_20260304_180000.md`

包含：

- 当前分支与 commit
- 关键配置快照（脱敏，不包含 API Key）
- 验收结果表（每一步 success/failed）
- `docker compose ps` 状态
- 各 smoke 输出原文
- 当前 git 工作区状态
- 建议提交命令

## 注意事项

- 脚本不会自动 `git add/commit/push`，只做记录与建议。
- 若当前 docker 环境不可用，compose 状态区块可能为空，不影响报告生成。
