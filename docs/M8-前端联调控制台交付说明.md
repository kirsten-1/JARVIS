# M8 前端联调控制台交付说明

## 1. 交付目标

在不影响现有 Spring Boot 网关后端的前提下，补充一个可直接联调的前端控制台，覆盖：

- 开发态 JWT token 签发
- 会话创建
- 同步聊天调用
- 流式聊天调用（SSE 增量展示）

## 2. 技术选型

- 框架：React 18
- 构建工具：Vite 5
- 语言：TypeScript
- UI：Ant Design 5

## 3. 代码结构

```text
frontend/
  index.html
  package.json
  tsconfig.json
  vite.config.ts
  src/
    main.tsx
    App.tsx
    styles.css
```

## 4. 已完成功能

- 提供 Base URL、UserId、Role、Provider、Model 的可视化输入。
- 支持调用 `POST /api/v1/auth/dev-token` 获取并展示 JWT。
- 支持调用 `POST /api/v1/conversations` 创建会话并自动回填 `conversationId/workspaceId`。
- 支持调用 `POST /api/v1/conversations/{id}/chat` 展示同步结果。
- 支持调用 `POST /api/v1/conversations/{id}/chat/stream` 并解析 `meta/delta/done` 事件。
- 支持手动停止流式请求（AbortController）。
- 支持调用运营指标接口并展示：
  - `GET /api/v1/metrics/overview`
  - `GET /api/v1/metrics/providers`

## 5. 本地启动

在 `gateway_pro/frontend` 下执行：

```bash
npm install
npm run dev
```

访问：`http://localhost:5173`

## 6. 联调验收步骤

1. 先启动后端 `GatewayApplication`（`dev` profile）。
2. 在前端填 `Base URL=http://localhost:8080`，`UserId=1001`，点击“获取 Dev Token”。
3. 点击“创建会话”，确认自动回填 `conversationId`。
4. 点击“同步聊天”，确认右侧“同步结果”出现回答。
5. 点击“流式聊天”，确认“流式结果”持续增长、`Meta` 区显示 `streamId` 信息。
6. 在“运营指标（M9.3）”卡片中点击“查询总览/查询 Provider”，确认表格数据可回显。

## 7. 当前限制

- 若 npm 源不可达，`npm install` 会失败（需本机可访问 `https://registry.npmjs.org`）。
- 目前前端仍以联调能力为主，尚未实现用户态登录与历史会话分页。
