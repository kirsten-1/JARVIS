# A15 基线趋势与漂移监控说明

## 目标

在 A11（单次基线）、A13（两报告差异）、A14（参考基线门禁）之后，补齐“多报告趋势分析”能力：

- 连续观察 P90 指标趋势，而不是只看单次结果
- 对核心指标给出 `pass/warn/fail` 结论
- 支持将趋势分析作为预发布门禁

## 交付内容

文件：

- `scripts/a15_baseline_trend.sh`

输出：

- `docs/reports/a15_baseline_trend_<timestamp>.md`

核心能力：

1. 聚合多份 A11 报告（默认 `docs/reports/a11_baseline_*.md`）。
2. 按指标提取 P90：`health`、`conversation_list_hot`、`message_list_hot`、`conversation_list_cold`、`message_list_cold`、`agent_loop`。
3. 基于窗口中位数评估最新漂移，给出 `improved/stable/regressed`。
4. 对核心指标（`health`/`conversation_list_hot`/`message_list_hot`/`agent_loop`）产出门禁结论：
   - `pass`：核心指标稳定或改善
   - `warn`：核心指标回归，但未超过阻断阈值
   - `fail`：核心指标回归超过阻断阈值

## 关键参数

- `REPORT_PATTERN`：报告匹配模式，默认 `docs/reports/a11_baseline_*.md`
- `MIN_REPORTS`：最少报告数，默认 `2`
- `WINDOW`：趋势窗口大小，默认 `7`
- `TREND_ROWS`：报告中展示最近样本数，默认 `10`
- `STABLE_PCT_THRESHOLD`：稳定区间阈值（%），默认 `8`
- `MAX_DRIFT_PCT`：核心指标最大漂移阈值（%），默认 `20`
- `EPSILON_MS`：微小抖动容差，默认 `0.05`
- `STRICT`：严格模式（`true` 时，`fail` 返回非 0）
- `FAIL_ON_WARN`：是否将 `warn` 当作失败
- `RUN_BASELINE_FIRST`：执行前先跑一次 A11（默认 `false`）

## 使用方式

## 1) 用默认报告目录分析趋势

```bash
./scripts/a15_baseline_trend.sh
```

## 2) 指定报告来源（例如临时目录）

```bash
REPORT_PATTERN='/tmp/a11_baseline_*.md' \
OUTPUT_REPORT='/tmp/a15_baseline_trend.md' \
./scripts/a15_baseline_trend.sh
```

## 3) 严格门禁模式

```bash
STRICT=true FAIL_ON_WARN=true ./scripts/a15_baseline_trend.sh
```

## 推荐流程

1. 先通过 A11 持续产出样本报告。
2. 用 A15 观察趋势是否稳定。
3. 关键节点继续用 A14 做 reference gate（阻断级门禁）。
