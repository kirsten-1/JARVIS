# A11 性能基线与回归报告说明

## 目标

建立可重复执行的性能基线脚本，用统一口径记录阶段性延迟指标，支撑：

- 版本回归对比（Axx 前后）
- 简历/复盘中的量化数据沉淀
- 发布前风险识别（阈值告警）

## 交付内容

## 1) 基线脚本

文件：

- `scripts/a11_baseline.sh`

能力：

- 自动完成认证（`TOKEN` / 本地 JWT / dev-token 三选一）
- 采集以下指标（默认 ms）：
  - `health`
  - `conversation_list_cold/hot`
  - `message_list_cold/hot`
  - 可选 `agent_loop`（`RUN_AGENT_BASELINE=true`）
- 自动生成报告：`docs/reports/a11_baseline_<timestamp>.md`
- 支持阈值检查与严格模式：
  - 默认仅告警（`warn`）
  - `STRICT=true` 时阈值不达标直接退出失败

## 2) 结果结构

报告包含：

- 指标统计表（Count / Avg / P50 / P90 / P99）
- 缓存收益估算（cold avg vs hot p50）
- 阈值检查结果（pass / warn）

## 使用方式

## 1) 默认执行（推荐）

```bash
./scripts/a11_baseline.sh
```

## 2) 启用 Agent 基线

```bash
RUN_AGENT_BASELINE=true \
AGENT_PROVIDER=aiservice \
AGENT_MODEL=MiniMax-M2.5 \
./scripts/a11_baseline.sh
```

## 3) 自定义阈值并启用严格门禁

```bash
STRICT=true \
TARGET_HEALTH_P90_MS=100 \
TARGET_CONV_LIST_HOT_P90_MS=220 \
TARGET_MSG_LIST_HOT_P90_MS=220 \
./scripts/a11_baseline.sh
```

## 常用参数

- `BASE_URL`：默认 `http://localhost:8080`
- `SAMPLES` / `WARMUP`：默认 `20 / 3`
- `RUN_AGENT_BASELINE`：默认 `false`
- `AGENT_SAMPLES` / `AGENT_WARMUP`：默认 `3 / 1`
- `REPORT_PATH`：可指定报告输出路径

## 注意事项

1. A11 是阶段性基线，不替代高并发压测工具（如 wrk/k6/JMeter）。
2. cold/hot 差异依赖读缓存策略，建议在同一环境下重复执行比较趋势。
3. 若目标是上线门禁，建议在 CI 中引入 `STRICT=true` 并固定运行环境。
