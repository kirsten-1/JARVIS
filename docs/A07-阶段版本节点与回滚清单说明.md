# A07 阶段版本节点与回滚清单说明

## 目标

为“关键阶段可记录、可提交、可回滚”提供标准化产物，满足：

- 阶段节点报告自动生成
- 提交信息模板自动生成
- 回滚清单固化到报告内
- 不自动执行 release，不自动 push

## 交付内容

- 脚本：`scripts/a07_stage_node.sh`
- 报告目录：`docs/reports/stages/`

## 使用方式

## 1) 生成阶段节点报告与 commit 模板

```bash
./scripts/a07_stage_node.sh A07-gateway-aiservice
```

生成文件：

- `docs/reports/stages/a07_stage_<timestamp>_<stage>.md`
- `docs/reports/stages/a07_stage_<timestamp>_<stage>.commitmsg.txt`

## 2) 建议提交动作（脚本只提示，不自动执行）

```bash
git add docs/reports/stages/a07_stage_<timestamp>_<stage>.md \
        docs/reports/stages/a07_stage_<timestamp>_<stage>.commitmsg.txt
git commit -F docs/reports/stages/a07_stage_<timestamp>_<stage>.commitmsg.txt
git push origin main
```

## 报告包含内容

- 分支/提交号/时间
- 当前关键配置快照（脱敏）
- 标准验收清单（m11/a04/a06）
- 路由回滚与镜像回滚命令清单
- compose 状态快照
- git 工作区快照

## 回滚策略说明

- 业务回滚优先级：
  1. 路由回滚（`AI_DEFAULT_PROVIDER` 切回直连厂商）
  2. 镜像回滚（`GATEWAY_IMAGE`/`FRONTEND_IMAGE` 回退至上一 tag）
- 回滚后统一执行：
  - `./scripts/m11_prod_up.sh`
  - `./scripts/m11_smoke.sh`

## 备注

- A07 是“阶段记录治理”，不是发布动作。
- 正式 release 仍建议在 JARVIS 全项目收官时统一执行。
